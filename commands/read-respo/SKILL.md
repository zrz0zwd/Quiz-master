# /read-respo — 学习出题风格

读取题库和参考书，分析出题风格并持久化存储到 agent 记忆中。

## 触发条件

用户输入 `/read-respo` 或说以下话语时触发：
- "学习出题风格"
- "导入题库风格"
- "分析某某出题方式"

## 执行步骤

### Step 1: 收集文件

向用户确认需要以下文件：

1. **题库文件** — 用于分析出题风格
   - 支持格式：`.md`（直接读取）、`.doc`/`.docx`（需通过 `/docx-editor-cn` 转换）
   - 内容应包含：题型、分值、考点、难度标注
   - 最少需要 3-5 道题才能有效分析

2. **参考书文件** — 用于后续出题的知识素材
   - 支持格式：`.md`、`.txt`、`.doc`/`.docx`

> 💡 如果用户已有之前学习过的风格记忆，告知用户并询问是否覆盖。

### Step 2: 读取文件

- 如果是 `.md` 或 `.txt`，直接读取内容
- 如果是 `.doc`/`.docx`，调用 `/docx-editor-cn` 转换为 Markdown

### Step 3: 分析出题风格

从题库中提取以下 8 个维度的风格特征：

| 分析维度 | 具体指标 |
|---------|---------|
| **题型结构** | 选择题/填空题/计算题/证明题/简答题的比例 |
| **分值分布** | 单题分值范围、总分设置、各题型分值占比 |
| **难度梯度** | 基础题→中等题→难题的比例（建议 5:3:2） |
| **考点覆盖** | 高频考点、冷门考点、跨章节综合题比例 |
| **表述风格** | 题干长度、专业术语密度、是否给提示/引导 |
| **数学表达** | 公式复杂度、是否需要推导、是否允许近似 |
| **物理背景** | 是否结合实际场景、是否需要建模 |
| **解题要求** | 是否需要画图、数值计算精度、单位要求 |

### Step 4: 写入记忆

将分析结果保存到 agent 记忆，键名为 `exam-style-{科目简称}`。

### Step 5: 确认完成

告知用户分析了多少道题、提取了哪些特征、下一步可以用 `/chaptest` 或 `/anology-test` 出题。

## 输出示例

```
✅ 出题风格学习完成！

已分析 12 道样题，提取以下特征：
- 题型结构：计算题 60% + 证明题 30% + 简答 10%
- 分值分布：单题 7-25 分，总分 150 分
- 难度梯度：基础 50% / 中等 30% / 难题 20%
- 高频考点：热力学函数、麦克斯韦关系、相变...

风格画像已保存至记忆：`exam-style-热统`

下一步：
- 用 `/chaptest 热统 第一章到第三章` 出章节测试
- 用 `/anology-test 热统 全范围` 出综合模拟
```
